# Prisma Migrations

## Purpose

This folder stores the database schema history for Prisma-managed changes.

## What Lives Here

- Timestamped migration directories generated by Prisma.
- Each migration directory typically contains a `migration.sql` file describing the concrete SQL change set.

## How It Fits Into The System

Migrations are how schema changes move from `schema.prisma` into real databases:

- local Docker/PostgreSQL
- backend e2e test database
- production RDS through the deploy workflow

## Concepts To Know

- The schema file describes the intended model.
- A migration records a concrete change from one schema state to the next.
- Keeping migration history committed is what lets new environments converge to the same database structure.

## Gotchas

- Do not hand-edit old migrations casually once they have been applied to shared environments.
- This folder is documented as a container concept. The timestamped migration leaf folders are generated artifacts and do not each get their own README.
